# 맡겨멍 — 기능 명세

이 문서만 읽어도 앱에 어떤 기능이 있는지 알 수 있도록 정리한 기능 카탈로그.
(작업 이력은 `PROGRESS.md`, 기술적 설계 배경은 `PORTFOLIO.md` 참고)

**맡겨멍**은 반려견을 맡길 수 있는 전국의 애견 유치원·호텔을 지도에서 탐색하고,
찜·리뷰·채팅·예약까지 한 앱에서 해결하는 iOS 플랫폼이다.
가게 데이터는 공공데이터(동물위탁관리업 API)에서 가져오고,
예약·채팅·리뷰·찜 등 사용자 데이터는 자체 백엔드(Cloudflare Workers + D1)에 저장한다.

**마지막 업데이트:** 2026-07-18

---

## 화면 지도

```
시작(StartView) ─ 역할 선택 + 카카오 로그인
   └─ 홈(HomeView) ─ 카카오 지도 + 가게 핀 + 최근 본 가게
        ├─ 가게 상세(StoreDetailView) ─ 리뷰·블로그 후기·찜·문의·예약
        │    ├─ 예약(BookingView) → 예약 완료(BookingDoneView)
        │    └─ 채팅방(ChatRoomView)
        ├─ 사이드바(FAB): 강아지 프로필 / 채팅 / 예약 내역 / 설정(마이페이지)
        │                 + 받은 예약 요청 · 알림장(사장님 전용)
        ├─ 채팅 목록(ChatListView) ─ 받은 문의(사장님) + 내 채팅
        ├─ 예약 내역(ReservationListView) ─ 확정 카드 '알림장 보기' → 알림장(DiaryTimelineView, 열람)
        ├─ 받은 예약 요청(OwnerModeView, 사장님 전용)
        ├─ 알림장 목록(OwnerDiaryListView, 사장님 전용) → 알림장(DiaryTimelineView, 작성/수정/삭제)
        └─ 마이페이지(MyPageView)
             ├─ 프로필 편집 시트
             ├─ 내 가게 등록/해제 시트(MyStoreSheet, 사장님 전용)
             ├─ 찜한 가게(FavoritesView)
             └─ 강아지 프로필(DogProfileView)
```

## 1. 계정 · 로그인

### 카카오 간편로그인
- 시작 화면에서 **역할(보호자 / 보호자·사장님)을 선택**한 뒤 카카오 로그인.
- 처음이면 자동 회원가입(`POST /api/auth/kakao`), 서버에 저장된 연락처·주소가 있으면 로컬 프로필(`UserProfile`)에 동기화.
- 로그인 창을 사용자가 직접 닫으면(취소) 에러 문구를 띄우지 않는다.

### 역할은 계정에 영구 귀속
- 최초 가입 시 선택한 역할이 카카오 계정에 묶인다(서버 `users.is_owner`).
- 같은 계정으로 **반대 역할 가입 시도는 서버가 409로 거절**하고, 앱이 "이미 ○○ 계정으로 가입된 카카오 계정이에요" 안내를 띄운다.
- 목적: 견주와 사장님이 채팅·찜·예약·최근 본 가게 데이터를 섞이지 않게 완전 분리. (탈퇴 기능은 미구현 — v2 예정)
- 역할에 따라 화면을 분기하지 않는다. 로그인 후 목적지는 항상 홈이고, 사장님 여부는 사이드바의 '받은 예약 요청' 노출과 마이페이지 '내 가게' 항목 등 **기능 단위로만** 달라진다.

### 개발자 진입 (시뮬레이터용)
- 카카오 SDK 없이 테스트할 수 있는 우회 로그인. 역할별 별개 계정(`dev-simulator` 견주 / `dev-simulator-owner` 사장님)이라 시뮬레이터 한 대에서 양쪽 시점을 모두 테스트할 수 있다.

### 로그아웃
- 마이페이지에서 확인 알림 후 로그아웃. 내비게이션 스택을 시작 화면으로 통째로 교체하고, 선택 상태·최근 본 가게 등 세션 데이터를 전부 비워 계정 간 잔존 데이터를 막는다.

## 2. 홈 · 지도 탐색

### 전국 가게 지도
- 공공데이터 **동물위탁관리업 API**로 전국의 위탁관리업체(유치원·호텔)를 불러온다.
- 좌표가 TM(EPSG:5181)이라 자체 구현한 `TMConverter`로 WGS84 위경도로 변환(대한민국 범위 밖 좌표는 필터링).
- 마스킹된 주소는 **카카오 로컬 API**로 보강.

### 뷰포트 기반 핀 표시
- 지도 이동이 멈추면 현재 화면에 보이는 위경도 범위 안의 가게만 핀으로 표시(도 경계 무관), 중심에서 가까운 순으로 개수 상한. 초기 화면은 수도권 뷰.

### 최근 본 가게
- 핀을 눌러 확인한 가게가 홈 하단 '최근 본 케어' 목록에 쌓인다.
- **계정별로 영구 저장**(UserDefaults `recent_pins_<userId>`) — 재로그인해도 유지되고, 최신순 최대 6개(넘치면 오래된 것부터 삭제), 견주/사장님 계정이 각자의 목록을 가진다.

### 사이드바(FAB)
- 홈 우하단 발자국 버튼: 강아지 프로필 / 채팅 / 예약 내역 / 설정(마이페이지).
- 사장님 계정에만 **받은 예약 요청** 항목이 추가로 노출된다.

### 종 아이콘 — 안 읽은 채팅 알림
- 홈 우상단 종 아이콘: **안 읽은 채팅이 1개 이상이면 빨간 점**, 모두 읽으면 사라진다. 탭하면 채팅 목록으로 이동.
- 기준: 내가 손님인 방들의 상대(가게·자동) 메시지는 모든 계정 공통이고, **사장님 계정은 내 가게로 온 손님 메시지를 합산**한다(사장님도 다른 가게에는 손님으로 예약·문의하므로 양쪽 다 반영). 채팅방을 열람하면 그 시점의 마지막 메시지까지 읽음 처리(`chat_room_reads`), 홈에 돌아올 때 갱신된다.

### 알림 피드 — 서버 폴링 + 로컬 알림
- 유료 Apple Developer 멤버십이 없어 원격 푸시(APNs)가 불가능해 대체 구현: 서버가 `GET /api/users/:id/notifications`로 "새로 생긴 일"을 opaque 커서 기반 증분 피드로 내려주고, iOS(`AppNotificationService`)가 이를 폴링해 로컬 알림(`UNUserNotificationCenter`)으로 발행한다.
- **알림 대상 4종** — 새 채팅·시스템 메시지(양쪽 계정), 새 예약 요청(사장님), 예약 확정(고객), 고객 취소(사장님).
- **폴링 주기** — 포그라운드는 앱 사용 중 30초 주기, 백그라운드는 `BGAppRefreshTask`가 iOS 재량으로 깨어날 때 같은 폴링을 1회 수행(실행 시점 보장 없음).
- **탭 딥링크** — chat 알림은 해당 채팅방, 예약 확정은 예약 내역(`ReservationListView`), 새 예약 요청·고객 취소는 받은 예약 요청(`OwnerModeView`)으로 이동.
- **알림 1회 원칙** — 각 이벤트는 채팅 축(이미 채팅 메시지가 남는 이벤트)과 예약 축(메시지가 없는 이벤트) 중 정확히 하나로만 피드에 들어가 중복 알림이 없다. 지금 열려 있는 채팅방과 같은 방의 chat 알림은 배너를 생략한다(이미 폴링으로 화면에 보이는 중).

## 3. 가게 상세

- 가게 이름·주소·타입(호텔🏨/유치원🏠) 카드.
- **주소 정밀 보강** — 공공데이터의 마스킹 주소(`***`)를 **네이버 지역 검색 우선**(층·건물·호까지 상세)으로 교체, 전화·카카오맵 링크는 카카오로 보강, 둘 다 없으면 좌표→지번 폴백. 동명 타지역 오매칭은 핀 좌표 3km 검증으로 차단.
- **찜(하트) 토글** — 서버 저장, 마이페이지 찜 목록과 연동.
- **내 가게 뱃지** — 로그인한 사장님이 등록한 가게면 '내 가게' 표시.
- **우리 아이 리뷰** — 이 가게에 남겨진 보호자 리뷰(평균 별점 + 보호자 수 + 태그). '리뷰 쓰기'로 작성 시트 진입.
- **블로그 후기** — 네이버 블로그 검색 결과 표시(NCP NAVER API HUB 경유, 2026-07-17 활성화).
- **맡기기 전 체크리스트** — 위탁 전 확인사항 안내.
- **문의하기(💬)** — 이 가게와의 채팅방으로 직행(방이 없으면 첫 전송 때 생성).
- **예약하기** — 예약 화면으로 이동.

## 4. 예약

### 예약 신청 (BookingView)
- **강아지 선택** — 등록해 둔 우리 아이 목록에서 선택(이름·나이·견종·몸무게·메모 요약 표시).
- **보호자 정보** — 연락처·주소 입력, '자동 불러오기'로 프로필 값 채움. 입력값은 다음 예약을 위해 영구 저장.
- **서비스 선택** — 시간권(시간당 ₩5,000 × 이용 시간), 종일권(₩35,000), 하루 숙박권(₩50,000). 픽업 서비스 포함 **총 결제 금액 실시간 계산**. 정기권은 가게 상세의 체크리스트에서 가게 문의로 안내한다. (날짜 선택보다 위에 배치 — 유형에 따라 시간 UI가 달라지므로 먼저 고른다.)
- **날짜·시간** — 캘린더(그래픽 DatePicker)에서 오늘 이후 원하는 날짜를 자유롭게 선택. **시간권만** 시작 시간(시간 피커로 자유 선택) + 이용 시간(1~6시간)을 고르고, 종일권·하루 숙박권은 날짜만 선택한다. 예약 문자열은 연도 포함 형식(시간권 "2026-07-12 (일) 14:00", 종일권·숙박권 "2026-07-12 (일)")으로 저장된다.
- **요청사항** — 알레르기·사료·산책 등 자유 입력.
- 신청 버튼 위에 **"가게 사정으로 예약이 취소될 수 있어요. 취소되면 채팅으로 알려드려요."** 경고 문구 표시.
- 신청 시 `POST /api/reservations` → 예약(REQUEST 상태)과 **채팅방이 자동 생성**되고, 완료 화면에 가게/강아지/일정/금액이 표시되며 바로 채팅방으로 이동할 수 있다.

### 기기 캘린더 자동 저장
- 예약 요청과 **동시에 고객 기기 캘린더에 일정이 추가**된다(제목 "가게명 예약", 메모 "강아지명 맡김"). 사장님 기기에는 저장하지 않는다.
- 삭제: 본인이 취소하면 즉시, 사장님이 취소한 경우엔 고객이 예약 내역을 열 때 CANCELED 상태를 감지해 삭제(서버 푸시 없음).
- 캘린더 권한이 거부돼 있으면(iOS는 시스템 알림을 다시 안 띄움) **"캘린더 권한이 꺼져 있어요" 알림으로 설정 앱 이동을 안내** — 예약 자체는 정상 처리.

### 예약 내역 (ReservationListView)
- 지금까지의 예약을 카드로 표시 — 가게명·일정·서비스 타입·상태(예약 요청됨/예약 확정/예약 취소됨), 호텔🏨/유치원🏠 아이콘.
- REQUEST/CONFIRMED 상태면 **예약 취소** 가능(확인 알림 + 낙관적 업데이트, 실패 시 원상복구).
- CONFIRMED 카드에는 **'알림장 보기'** 버튼 — 그 예약의 알림장 타임라인을 열람(아래 참고).

### 알림장 (DiaryTimelineView)
- **예약당 타임라인** — 사장님이 맡은 아이의 하루("초코가 친구들과 잘 놀았어요")를 글로 남기고, 보호자가 그 예약 단위로 열람한다.
- **사장님(작성):** 홈 사이드바 '알림장' → 맡은 아이들(확정 예약) 목록(`OwnerDiaryListView`) → 작성/수정/삭제(`DiaryComposer` 시트).
- **보호자(열람):** 예약 내역 확정 카드 '알림장 보기' → 같은 화면을 읽기 전용으로 사용(`diaryContext.canWrite` 플래그로 시점 분리 — 채팅 말풍선과 같은 패턴).
- 사장님이 **작성할 때만** 보호자 채팅방에 자동 메시지(sender 0) 발송 — 안 읽은 채팅 빨간 점에 자동 반영. 수정·삭제는 메시지 없음.
- 사진 첨부는 R2(이미지 저장소) 활성화 후 추가 예정 — `diary_photos` 스키마만 준비, API는 항상 `photos:[]` 반환.

### 예약 확정/취소 (사장님, OwnerModeView)
- '받은 예약 요청' 화면에 **내가 등록한 가게로 온 REQUEST만** 표시 — 카드에 가게명, "강아지 : 이름"과 "예약자 : 이름" 두 줄, 일정·서비스 유형.
- **카드 탭 → 상세 시트**(화면 이동 없음) — 강아지 정보(이름·견종·나이·몸무게·성별·메모), 예약자 정보(이름·연락처·주소), 예약 정보(가게·일정·유형·요청사항). API가 `pets`·`users` 조인으로 함께 내려준다.
- **확정하기** → 고객 예약 내역에 '예약 확정'으로 표시.
- **예약 취소** → 확인 알림 후 취소(`by_owner` 플래그), 고객에겐 '예약 취소됨'으로 표시되고 고객 캘린더 일정도 동기화 삭제된다. 고객 채팅방에는 **"가게 사정으로 ○○ 예약(일시)이 취소되었어요" 자동 메시지**가 발송된다(고객 본인 취소 시에는 미발송).
- 내 가게를 아직 등록 안 했으면 빈 화면에 등록 안내 문구.

## 5. 채팅

### 손님 ↔ 사장님 양방향 채팅
- 채팅방은 **(손님 + 가게) 조합당 1개**. 사장님용 방을 따로 만들지 않고 같은 방에 각자의 `sender_id`로 참여한다 — 말풍선 방향은 "내가 보낸 메시지인가" 기준이라 양쪽 시점 모두 자연스럽다.
- 예약하면 방이 자동 생성되고, 가게 상세 '문의하기'로도 진입 가능(방이 없으면 첫 전송 때 생성).

### 채팅 목록 (ChatListView)
- **받은 문의** 섹션(사장님 전용) — 내가 등록한 가게로 온 손님들의 문의방 목록(손님 닉네임 + 가게명 + 마지막 메시지). 보호자 프로필은 강아지 아바타(dog_b/dog_c, 방마다 고정 순환)로 표시.
- **내 채팅** 섹션 — 내가 손님으로서 가게에 보낸 채팅방 목록. 가게 프로필은 타입별 아이콘(유치원 kindergarden / 호텔 hotel)으로 표시.

### 채팅방 (ChatRoomView)
- 실제 메시지 로드/전송. 전송은 낙관적 추가(실패 시 롤백), 방을 열어두는 동안 **3초 간격 폴링**으로 상대 메시지 자동 반영.
- 상단·상대 말풍선 프로필은 채팅 목록과 동일 — 손님 시점은 가게 타입 아이콘(유치원/호텔), 사장님 시점은 보호자 강아지 아바타(`AppRouter.chatRoomAvatar`, 진입점마다 설정).
- 방을 열람하면(최초 로드 + 보는 중 도착분) 마지막 메시지까지 **읽음 처리** — 홈 종 아이콘 빨간 점의 기준이 된다.
- 예약 시 "예약 요청이 완료되었습니다" 등 **자동 메시지**(sender 0 — 예약 취소 안내·리뷰 요청·알림장 알림 포함)가 방에 남는다 — 말풍선이 아니라 화자 없는 **중앙 정렬 시스템 안내문**(작은 글씨 + 크림색 칩)으로, 손님·사장님 양쪽 시점 동일하게 표시.
- **리뷰 요청 자동 메시지** — 확정된 예약의 이용일 다음날 오후 6시(KST), 서버 Cron이 그 채팅방에 "어제 ○○ 이용은 어떠셨나요? 리뷰를 남겨주세요" 메시지를 자동 발송(예약당 1회, 연·월·일 정확 대조).
- **예약 취소 자동 메시지** — 사장님이 예약을 취소하면 그 즉시 고객 채팅방에 "가게 사정으로 ○○ 예약(일시)이 취소되었어요" 메시지를 자동 발송(고객 본인 취소는 미발송).
- 상단 '응답중' 상태 표시는 손님 시점에서만 노출(사장님 시점에선 숨김).

## 6. 리뷰

- **보호자 펫 리뷰** — 별점 + 태그(예: 시설이 깨끗해요 등, 서버 관리 태그 목록) + 텍스트로 작성(`ReviewWriteSheet`), 가게 상세에 평균 별점·보호자 수·태그와 함께 표시. API는 `pet-reviews` 계열.
- **네이버 블로그 후기** — 가게 이름으로 네이버 블로그를 검색해 제목·날짜를 보여주는 기능. NCP NAVER API HUB 키 연동으로 **활성화**(2026-07-17).

## 9. 마이페이지

- **프로필 카드** — 이름 + 실데이터 통계(예약 수·강아지 수·채팅 수).
- **프로필 편집** — 이름·연락처·주소 시트 편집. 로그인 상태면 서버(`PUT /api/users/:id`)에도 저장 + 세션 닉네임 동기화.
- **예약 내역 / 찜한 가게 / 채팅** — 각 화면으로 이동(개수 뱃지 표시).
- **내 가게** (사장님 전용) — `MyStoreSheet`에서 공공데이터 가게를 **상호명 부분일치 검색**(2글자+, 최대 30건)해 등록. 남의 가게면 409 거절. 등록된 가게 목록에서 **해제**(잘못 등록 복구)도 가능. 등록한 가게 수/이름이 뱃지로 표시된다.
- **우리 아이 프로필** — 강아지 프로필 화면으로 이동.
- **로그아웃** — 확인 알림 후 로그아웃.
- UI만 있고 동작 미연결: 쿠폰함, 알림 설정, 앱 설정, 1:1 문의, 공지사항.

## 12. 핵심 데이터 규칙 (설계 불변식)

1. **공공 가게의 ID 승격** — 지도 가게는 DB id가 없으므로 `store_key`(`"이름|주소"`)로 `stores`에 upsert. 같은 가게는 항상 같은 `store_id`가 된다. 예약·찜·리뷰·채팅·내 가게 등록이 전부 이 규칙 위에 있다.
2. **채팅방·찜은 (사용자+가게)당 1개** — 예약마다 방을 만들지 않는다.
3. **역할은 계정 귀속** — 카카오 계정 하나 = 역할 하나(최초 가입 시 결정, 서버 409로 강제).
4. **캘린더는 고객 기기에만** — 추가는 예약 요청 시점, 삭제는 본인 취소 즉시 / 사장님 취소는 예약 내역 열 때 동기화.

## 13. 미완성 · 데모 범위

- **알림장 사진 첨부** — R2(이미지 저장소) 미활성으로 보류. `diary_photos` 스키마와 API 계약(`photos:[]`)만 준비 — 활성화 시 업로드/서빙 엔드포인트와 `PhotosPicker`만 추가하면 된다.
- **일부 가게 주소 마스킹 잔존** — 네이버·카카오 어디에도 등록되지 않은 4곳은 DB에 마스킹 주소 유지(추측으로 채우지 않음) — 가게 상세 화면에서는 좌표→지번 폴백으로 최소한의 주소 표시.
- **회원 탈퇴** — 미구현(역할 잘못 가입 시 복구 수단, v2 예정).
- **마이페이지 일부 항목** — 쿠폰함(뱃지 "5장" 고정)·알림 설정·앱 설정·1:1 문의·공지사항은 UI만 존재.
- **원격 푸시 없음** — 유료 Apple Developer 멤버십 미보유로 APNs 불가. 화면 안은 채팅 3초 폴링·예약 상태는 화면 진입 시 조회로 반영, 화면 밖은 §2 "알림 피드"의 서버 폴링 + 로컬 알림으로 대체(실시간 즉시성은 아님).
- **API 인증 없음** — 모든 라우트가 클라이언트가 보낸 `user_id`를 신뢰한다(토큰 없음). 앱 화면 흐름상 계정 간 데이터가 섞이지 않는 것은 검증됐지만(2026-07-12 전수 점검), API를 직접 호출하면 타인 데이터 접근이 가능한 포트폴리오 데모 범위 — 취소/확정·펫 삭제 등에 소유 검증 없음. 실출시 시 JWT 등 인증 레이어 필요.
